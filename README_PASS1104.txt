Pass1104 adds Editor.Ada_Tagged_Derived_Legality as a widened semantic-progress pass for tagged, derived, private-extension, interface, overriding, abstract-operation, dispatching-call, and class-wide conversion legality.

The new package is snapshot-owned and fixture-friendly. It consumes existing assignment legality, return legality, and dispatching-call legality models plus resolved tagged/derived semantic facts supplied by callers. It performs no parsing, file IO, buffer mutation, dirty-state mutation, command/keybinding/workspace mutation, render mutation, compiler invocation, LSP integration, external parser generation, Python integration, or shell-script integration.

The legality classifier covers legal tagged derivation, private extension, interface derivation, primitive operation inheritance, overriding declarations, abstract type requirements, dispatching calls, and class-wide conversions. It also classifies unresolved/non-tagged parents, limitedness mismatches, private/limited view barriers, missing or profile-mismatched interface operations, duplicate inherited primitives, missing or malformed overrides, abstract operations not overridden, nonabstract types that still have abstract operations, unresolved/ambiguous/non-dispatching dispatching targets, missing controlling operands, ambiguous controlling results, class-wide conversion incompatibility, and linked assignment/return legality errors.

Regression coverage is in Test_Ada_Tagged_Derived_Legality_Pass1104 and is registered in Core_Suite. The regression exercises derivation/interface/private-view classification, overriding conformance errors, abstract-operation enforcement, dispatching controlling-operand errors, class-wide conversion incompatibility, deterministic counters, lookups, and fingerprints.

This pass intentionally avoids another IDE projection chain layer. It adds one compiler-grade semantic building block for tagged/derived/private/interface operation legality. Full compiler-grade Ada analysis remains incomplete until the remaining overload resolution, type checking, generic contracts, freezing/representation legality, and cross-unit semantic closure layers are fully integrated.
